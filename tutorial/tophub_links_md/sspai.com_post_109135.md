Maestri：AI Agents 时代的白板工具

平台：macOS

关键词：AI、Agent、终端

@Vanilla

：说实话，当我打开 Maestri 官网的时候，我以为这又是一款强行蹭 AI 概念的应用。不过，当我真的下载安装并用它来完成了一个小项目时，我发现 Maestri 在一些特定场景下还真的挺不错，可以帮助我们更好地借助 AI Agents 来完成一些项目。接下来，我就以我最近在做一个投资调研项目为例，来看看 Maestri 究竟能做什么。

打开 Maestri，你就能发现它的主界面就是一个白板画布，你可以添加终端、标签、附件、文件树、门户、格式、绘画等元素，其中文件数可以看作是文件夹，门户就可以看作网页，格式可以看作文本框，其他的元素就顾名思义了。

在画布顶部的工具栏中，除了前面提到可以添加的元素，还有一个选择按钮，可以选中元素来移动或者调整尺寸。所有在工具栏上的按钮都有对应的键盘快捷键，一般都是一个字母按键，可以非常快速地调用。

再看到画布的右下角，总共有三个按钮，分别是楼层、小地图和缩放。

楼层功能是 Maestri 利用 macOS 系统特性打造的一个功能，它可以创建多个楼层，相当于多个独立的工作区，在创建的时候可以新建也可以直接复制底层，最后可以 merge，也可以将其中一个楼层作为最终落地层，有点 Git 的味道。

小地图就是整个画布的缩略图，可以帮助我们快速地定位到大致的区域。缩放功能就更加简单了，通过加号和减号按钮来控制缩放比例即可。

目前，Maestri 的终端支持了 Claude Code、Codex、Gemini CLI、OpenCode 和 Shell，可以在插入终端元素的时候进行选择。

当我们在画布中插入多个终端时，可以通过连接线连接，让各个终端之间可以互相通信。除了终端之间的连接，我们还可以让标签、门户连接到终端元素上，让 AI Agents 读取、编辑这些元素中的内容。

Maestri 其实是通过 Maestri Skill 来让 AI Agents 进行互联通信，以及操作连接的标签和门户，所有只能限制于这个 Maestri Skill 包含的操作，比如说无法让 AI Agents 新建便签并写入内容，也无法和文件树、文本框等元素建立链接。

综合来讲，Maestri 解决了多个 Agents 协作时频繁切换、数据不互通等问题，通过「画布」设定让项目中的多媒体文件可以同时展示出来，特别是在大屏设备的加持下，可以大大提高我们阅读内容的效率。
