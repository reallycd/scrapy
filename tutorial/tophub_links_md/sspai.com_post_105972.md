OpenClaw：个人数字助理 AI 工具

平台：macOS / Windows / Linux

关键词：AI、个人助理

@Vanilla

：最近一周，本地自部署 AI 工具 OpenClaw（最初的名字叫 Clawdbot，后来又改名为 Moltbot）属实是狠狠刷屏了，甚至掀起了一股购买 Mac mini 的热潮。与传统的 Chatbot 不同，OpenClaw 最大的优势在于三个方面：

隐私可控、模型自由、调用方便

。

在本期派评中，我不想展开讲解枯燥的参数或者原理，而是更想和大家聊一聊 OpenClaw 到底是什么，以及它可以为你的数字生活做些什么。

OpenClaw 并非一个单纯的大语言模型，也不是一个单纯的 Coding CLI，它其实是一个本地的「数字生命中枢」。它集成了 Model、Agent、Skill、MCP 等模块化架构，通过驻留在 Mac mini 或服务器上的 Gateway 实现跨平台的接入与异步调度。无论身处何地，你都能通过 各种即时通讯工具与之即时互动。

与此同时，所有在 OpenClaw 中产生的记忆、文件索引和个人习惯都储存在你自己的本地工作区（Memory.md、索引数据库、Skills 脚本等）中，官方宣称你可以完全掌控自己的数据主权，无需担心隐私泄露。

如果你手头上正好有一台 Mac mini，那自然是最好的选择，因为这相当于你拥有一台 7x24 小时低功耗运作的私人服务器，且能无缝接入 Apple 生态相关的 Skill。如果你没有 Mac mini，也无需特意购买，腾讯云、阿里云、Cloudflare、Digital Ocean 等云服务商均推出了 OpenClaw 专属云服务器镜像，可以非常方便地完成一键部署。

接下来，我来分享两个我近期使用 OpenClaw 的真实应用场景。

以往我们进行 Vibe Coding，往往需要守在 Claude Code、Codex、Gemini CLI 或 Antigravity 等桌面客户端前。而现在，我只需在手机上给聊天软件中的 Bot 发出指令，OpenClaw 就会在服务器端自动完成所有逻辑编写与调试，并实时传回结果。

比如最近我将 iPhone 上的主力输入法换成了「仓输入法」，但对默认皮肤不甚满意。我找了一个现成的 hskin 皮肤文件直接丢给 Bot，让它根据我的审美偏好和输入习惯对皮肤代码进行精准调整。调整完成后，我直接在手机上导入生成的皮肤文件即可，整个过程没动过电脑键盘。

我在电脑本地创建了一个包含源文件和索引的个人知识库，并配套了管理脚本。通过 OpenClaw，我可以远程对知识库下达指令：无论是同步新入库的文件索引，还是调用 Python 进行深度的全文检索、分析甚至辅助写作，都能在移动端瞬间完成。这样一来，我既享受到了大模型的理解能力，又无需将私有数据库上传至云端服务，真正实现了「数据本地，算力随身」。

对于 OpenClaw 的介绍我们先止步于此，后续我会单独出一篇更详细的文章来和大家一起探讨。我觉得 OpenClaw 最成功的地方不在于某种颠覆性的技术变革，而在于它精准捕捉了用户的真实需求，通过 Gateway 和 Channel 极大地拓宽了 AI 的使用边界。
