Vibe Island：灵动岛上的 AI Agent 助手

平台：macOS

关键词：灵动岛、AI Agent

@Vanilla

：在 Vibe Coding 时代，能实现各种功能的产品变得不再稀缺，从 App Store 今年巨幅增长的应用审核量就可以看出应用程序的创造几乎已经是一件零门槛的任务。然而，有品味的应用依旧是这个世界上的「艺术品」，像我这种没有编程基础和设计基础的人还是无法创造出一款有品味的产品。

在我看来 Vibe Island 就是这样一款有品味的作品，虽然利用 MacBook 的刘海区域来显示和交互 AI Agent 不是什么新鲜事，市面上也已经有了开源免费的产品，但是开发者 Edward Luo 无疑将 Vibe Island 打造成了一款设计精美、功能丰富的优秀作品，让其在一众竞品中脱颖而出。

Vibe Island 安装后会通过 hook 自动建立与本地 Agent 或者终端应用的连接，无需任何设置或者部署就可以直接开启使用。Vibe Island 的功能主要可以分为四个部分：监控、授权、询问和跳转。我们一项一项分别来看。

在监控部分，Vibe Island 可以显示当前 session 的标题和任务概览，帮助我们通过灵动岛快速获取 AI Agent 的任务进行情况。Vibe Island 目前支持了 Claude Code、Codex、Gemini CLI、Cursor、OpenCode、Droid、Qoder、Copilot、CodeBuddy、Kiro 共十款 AI Agent 以及 iTerm2、Ghostty、Warp、Terminal.app、VS Code、Cursor 等 13 款终端应用，都可以在 Vibe Island 的灵动岛中进行区分显示，帮助我们清晰地区分多线程任务。

在授权部分，Vibe Island 可以在灵动岛中直接显示当前 session 的权限选择窗口，包括拒绝、允许一次、全部允许、自动批准四个选项，而且你也可以通过对应的键盘快捷键进行快速确认。这个模块带来的便利有两个，一是授权通知可以让我们第一时间进行权限选择，避免了任务进度被长时间暂停，另一个则是无需打开终端应用即可完成权限选择，不会影响当前正在进行的活动。

在询问部分，效果与授权部分类似，我们可以直接在灵动岛对 AI Agent 提出的问题进行选择并提交，帮助 AI Agent 根据我们的意愿完成接下去的任务，也是减少了与终端应用的不必要交互。

最后的跳转部分，则是指我们可以点击 Vibe Island 在灵动岛上的任一项，它都会直接跳转到终端应用的相应 session 中，以便让我们查看 session 详情或者输入指令。

在设置中，Vibe Island 提供了一些必要的选项。显示方面，我们可以更改刘海的样式，包括简介和详细两种选项；面板中，可以调整内容字体大小、完成卡片高度和最大面板高度；对于 AI Agents，我们也能启用或者关闭显示代理活动详情。

在声音和快捷键设置面板中，我们可以看到 Vibe Island 页提供了诸多选项，可以满足我们个性化的需求。
